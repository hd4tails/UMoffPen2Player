# UMoffPen2Player Rules

このリポジトリは、UMoffPen2のセーブデータをブラウザだけで再生する静的Webツールとして扱う。

## Product Scope

- UMoffPen2PlayerはUMoffPen2のセーブデータを再生するツール。
- 現在のUMoffPen2セーブデータ形式を対象にする。
- 古いセーブデータ形式との互換性は、明示的な依頼があるまで不要。
- 依存パッケージや外部CDNを追加しない。基本は単一の静的HTMLで動作させる。
- セーブデータはブラウザ内で処理し、サーバーへ送信しない。

## License And Publication

- 権利者表示は `はい`、英語表記が必要な場所では `hy` とする。
- ライセンスはオープンソースではなく、公式公開ページでの利用のみ許可する独自ライセンスとする。
- UMoffPen2本体、UMoffPen2の素材、マニュアル、パッケージ、prefab、ユーザーが作成したセーブデータは、このリポジトリのライセンス対象に含めない。
- 公開前に、個人情報、ローカルパス、別プロジェクト名、秘密情報、不要なTODO/FIXME/HACK/debug文言が残っていないか確認する。

## UI Rules

- ヘッダー説明文は `UMoffPen2のセーブデータを再生するツールです。` とする。
- 入力欄ラベルは `セーブデータ` とする。
- `.txt を読み込み` はセーブデータ入力欄の直下に置き、`再生` / `クリア` ボタンはファイル選択欄の下に置く。
- 画像欄には `画像` タイトルを置く。
- タイムラプス欄には `タイムラプス` タイトルを置く。
- ステータス欄には `ステータス` タイトルを置く。
- ステータスの統計は、データサイズ、レイヤー数、コマンド数、再生時間、ストローク数を表示する。
- 展開バイト数など、内部実装寄りの詳細は通常UIに表示しない。

## Preview And Rendering

- ライブプレビューは常に有効にする。
- ライブプレビュー間隔は30fps固定とする。
- PNGや動画用の描画はUMoffPen2標準の1920x1080を基準にする。
- 動画解像度を下げる場合も、描画自体は1920x1080で行い、最後に動画サイズへ縮小する。

## Image Export

- 通常PNG保存は `PNGで保存` ボタンで行う。
- レイヤー別保存は個別ボタンを並べず、`レイヤーごとに保存` ボタン1つにする。
- レイヤー別保存は各レイヤーPNGをZIPにまとめて保存する。
- レイヤーPNGのファイル名は次の形式にする。
  `UMoffPen2Player_render_1920x1080_LayerX_Mode_a0.00.png`
- `Mode` は英語の合成モード名にする。
- `a0.00` はそのレイヤーのアルファ値を表す。

## Timelapse Export

- タイムラプスは指定した秒数の30fps MP4/H.264として作成する。
- 解像度は `1/1 - 1920x1080`、`1/2 - 960x540`、`1/4 - 480x270` から選べるようにする。
- MP4/H.264エンコードはブラウザの組み込み `MediaRecorder` を使う。H.264エンコーダーを同梱しない。
- 画像枚数が必要フレーム数より少ない場合は、同じ画像を複数フレームに使う。
- タイムラプスの最後のフレームには、必ず最終描画状態を入れる。
- 動画は透明を扱えないため、透明部分は `#FFFFFF` 背景に合成してから動画化する。

## Obfuscation

- 公開する `index.html` のJavaScript本体は難読化された状態を維持する。
- 難読化は秘匿ではなく、公開コードを読みづらくするためのものと扱う。
- 復号・再エンコード用の一時スクリプトや復号済みJSはコミットしない。
- 難読化後もページ読み込み、主要ボタンの初期状態、コンソール警告/エラーなしを確認する。

## Git And Deployment

- GitHub Pagesの公開URLは `https://hd4tails.github.io/UMoffPen2Player/`。
- GitHub上の履歴は、必要に応じて現在の公開状態だけの単一コミットへ置き換える。
- 履歴を維持しない運用を続ける場合、変更は現在の単一コミットへ `git commit --amend` し、`git push --force-with-lease origin main` で反映する。
- force push前には、作業ツリーが意図した差分だけであることを確認する。

## Verification

- 変更後は少なくとも次を確認する。
  - `git diff --check`
  - 難読化ローダーのJavaScript構文
  - ローカルHTTPでのページ初期化
  - コンソール警告/エラーなし
- 動画やダウンロード周りを変更した場合は、対象ボタンの有効/無効状態とステータス表示も確認する。
